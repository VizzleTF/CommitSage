# Telemetry

Commit Sage collects anonymous usage data to understand how the extension is used and to improve it over time. No personal data, file contents, diff contents, or API keys are ever collected.

---

## Disabling Telemetry

Telemetry can be disabled in two ways:

**VS Code global setting** — disabling telemetry in VS Code (`telemetry.telemetryLevel`) automatically disables it for all extensions including Commit Sage.

**Extension setting** — set `commitSage.telemetry.enabled` to `false` in your VS Code settings or in `.commitsage/config.json`:

```json
{
  "telemetry": {
    "enabled": false
  }
}
```

Both settings are respected independently — if either is off, no data is sent.

---

## What Is Collected

Every event includes the following base properties:

| Property | Example | Description |
|----------|---------|-------------|
| `vsCodeVersion` | `1.95.0` | VS Code version |
| `extensionVersion` | `2.5.1` | Commit Sage version |
| `platform` | `win32`, `darwin`, `linux` | Operating system |
| `environment` | `desktop`, `web` | VS Code environment type |

In addition, a random machine ID generated by VS Code (`vscode.env.machineId`) is used to deduplicate events. This ID is not linked to any account or personal identity.

---

## Events

### `extension_activated`

Fired once when the extension starts.

_No additional properties._

---

### `extension_deactivated`

Fired once when VS Code closes or the extension is disabled.

_No additional properties._

---

### `message_generation_started`

Fired when the AI commit message generation begins.

| Property | Type | Description |
|----------|------|-------------|
| `provider` | `string` | Active AI provider (`gemini`, `openai`, `codestral`, `ollama`) |
| `diffSize` | `number` | Size of the diff in characters (before truncation) |
| `fileCount` | `number` | Number of changed files |
| `truncated` | `boolean` | Whether the diff exceeded the 100 000-character limit and was truncated |

---

### `message_generation_completed`

Fired when the AI successfully returns a commit message.

| Property | Type | Description |
|----------|------|-------------|
| `provider` | `string` | AI provider used |
| `model` | `string` | Exact model name returned by the provider |
| `durationMs` | `number` | Time from generation start to completion, in milliseconds |
| `language` | `string` | Configured commit language (`english`, `russian`, `custom`, etc.) |
| `onlyStagedChanges` | `boolean` | Whether only staged changes were analyzed |

---

### `message_generation_failed`

Fired when generation fails for any reason (API error, network issue, etc.).

| Property | Type | Description |
|----------|------|-------------|
| `provider` | `string` | AI provider that was used |
| `error` | `string` | Sanitized error message (file paths are replaced with `<path>`) |
| `errorType` | `string` | Error class name (e.g. `Error`, `OpenAIError`, `ApiKeyInvalidError`) |

---

### `commit_completed`

Fired after a successful `git commit` (only when auto-commit is enabled).

| Property | Type | Description |
|----------|------|-------------|
| `hasStaged` | `boolean` | Whether staged changes were present |
| `hasUntracked` | `boolean` | Whether untracked files were present |
| `hasDeleted` | `boolean` | Whether deleted files were present |
| `messageLength` | `number` | Length of the generated commit message in characters |

---

### `commit_failed`

Fired when `git commit` fails (only when auto-commit is enabled).

| Property | Type | Description |
|----------|------|-------------|
| `error` | `string` | Sanitized error message |
| `errorType` | `string` | Error class name |

---

### `push_completed`

Fired after a successful `git push` (only when auto-push is enabled).

_No additional properties._

---

### `push_failed`

Fired when `git push` fails (only when auto-push is enabled).

| Property | Type | Description |
|----------|------|-------------|
| `error` | `string` | Sanitized error message |
| `errorType` | `string` | Error class name |

---

### `settings_changed`

Fired when any `commitSage.*` setting is changed in VS Code or `.commitsage`.

| Property | Type | Description |
|----------|------|-------------|
| `setting` | `string` | Name of the changed setting, without the `commitSage.` prefix (e.g. `provider.type`, `commit.commitLanguage`) |

---

## Error Sanitization

All error messages sent in telemetry events are sanitized before transmission: absolute file paths (e.g. `/home/user/project/src/file.ts` or `C:\Users\...`) are replaced with `<path>`. Stack traces are never included.
